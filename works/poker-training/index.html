線 A｜教學作品

為有底子的德州撲克玩家設計的 AI 訓練系統

兩份書面指南，用牌桌的語言，把 AI 從「問答工具」教成一套可累積的訓練系統。

對象
有底子的德州撲克玩家
（數年線上＋業餘賽經驗）
形式
兩份書面指南
入門篇＋進階篇
教材定位
給地圖，不給導航
設計核心
全程用對方的
牌桌語言

為什麼做這套教材

對方是有底子的玩家。用過 GTO Wizard、打過線上賽、懂 Range 和 SPR，但進步卡住了。復盤看數字，知道「應該 bet 63%」，下次上桌還是打一樣的錯。

他的問題不是不夠努力，是 GTO 教平衡打法，不教怎麼針對眼前這個人。而 Exploit 策略需要的對手情報、漏洞追蹤、概念內化，剛好全是 AI 擅長的事。他對 AI 完全陌生，所以教材要從零講起，但內容不能是新手等級。

設計思路

用他已經會的東西當橋。他是卡牌遊戲出身，所以我用爐石的 Metagame、對手 profile、資源管理來對應撲克的 Exploit 邏輯。新工具接在舊知識上，學習成本砍掉一大半。

Chat 和 Code 的差異，用教練來解釋。技術上這是 context 與檔案系統的差異，但對玩家來說，Chat 版是「每次去找不同的教練，從頭介紹一遍自己」，Claude Code 是「一個常駐教練，記得你三個月前河牌圈常犯什麼錯」。一句話：Chat 適合問一個問題，Code 適合經營一套系統。

拆成兩份，對應兩個階段。入門篇零安裝，打開瀏覽器就能開始用 AI 做 Exploit 分析和對手 profiling。進階篇才進 Claude Code，建對手檔案庫、漏洞追蹤、賽後復盤自動化。先讓他嚐到甜頭，再邀請他多走一步。

給地圖，不給成品。進階篇我刻意不幫他把系統建好，只畫出八個可能性方向，每個方向附一句「你可以這樣問 Claude」。讓 AI 當他的施工隊，他只需要想清楚要什麼。這本身就是在示範正確的 AI 使用姿勢。

把我踩過的坑先標出來。用了一個多月 Claude Code 的真實教訓：session 不要混用、大目標要切小塊、八二法則是 80% 用系統而不是建系統。坑不替他填，但先告訴他坑在哪。

教材架構

入門篇：用 AI 當你的撲克教練

  • 診斷：你需要的不是更多復盤，是 Exploit、概念內化、對手追蹤
  • 三個核心方向：Exploit 策略、對手 Profiling、新概念快速攝入
  • 完整工具箱：對話式手牌復盤、Range 思考訓練、對手模擬對打、Tilt 管理
  • AI 是什麼不是什麼：它是 GTO Wizard 的解說員，不是替代品
  • 零基礎起步：三步建好一個 Poker Coach 專案，附可直接複製的設定
  • 限制說明：賽中使用是紅線

進階篇：Claude Code 靈感大禮包

  • Chat 與 Code 的本質差異：問一個問題 vs 經營一套系統
  • 三種安裝方式比較，附針對他使用場景的明確建議
  • 可能性地圖八個方向：對手情報庫、CLAUDE.md 個人化教練、自動化復盤指令、手牌歷史數據分析、針對性訓練、個人戰術手冊、Tilt 日誌
  • 老司機踩坑 Tips：八條真實教訓
  • 我自己的系統長什麼樣：證明這條路走得通
教材節錄｜漏洞追蹤：把模糊感覺變成可查的記錄

不是「我知道我 river 打得不好」這種模糊感覺，而是「我在 BTN vs BB SRP，Turn 第二條街 barrel 時，持有弱 draw 的 bluff 頻率偏高，最近 5 次有 3 次被抓」這種具體記錄。因為 Claude 能讀這些檔案，你可以跟它說「看一下老王的檔案，幫我出今晚針對他的策略」，它就能給你針對性的建議。

教材節錄｜Solver 給數字，AI 給邏輯

Solver 告訴你 bet 幾 %，AI 告訴你為什麼這個 board 上你的 AQ 在 river 有 range 優勢，以及對這個 passive 對手應該偏離 GTO 往哪個方向。Solver 假設對手打 GTO，AI 可以假設對手打得很爛。這是 solver 做不到的事，也是 Exploit 策略真正的主戰場。

這份文件是地圖，不是導航。地圖告訴你有哪些路，但走哪條、走多快，都是你的事。

這堂課教會我的事

客製化教學的核心不是改例子，是換語言。當 Exploit 用爐石講、累積系統用教練講，對方根本不覺得自己在學 AI，他覺得自己在升級牌技。

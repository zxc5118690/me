線 A｜教學作品

為職業德州撲克玩家設計的 AI 訓練系統

對象是一位世界賽亞軍。兩份書面指南，用牌桌的語言，把 AI 從「偶爾問一下的網頁工具」，變成一個真正懂他的職業教練。

對象
職業德州撲克玩家
（前爐石戰記職業選手、世界賽亞軍）
形式
兩份書面指南
入門篇＋進階篇
教材定位
給藍圖，不蓋房子
設計核心
用對方的牌桌語言
全程客製化

為什麼做這套教材

我這位朋友的牌齡，是從《爐石戰記》開始的。他打過職業，拿過世界賽亞軍。後來他把卡牌玩家那套讀對手、抓節奏的邏輯，整個搬進德州撲克的圈子，成了一名專打線上賽的職業玩家。

有一次聊天，我問他：現在 AI 這麼強，你有沒有拿來精進、拿來復盤？他說有，但用起來「沒有大家講的那麼神」。他開著免費的網頁版，在同一個對話框裡丟牌局，AI 偶爾能給點概念回饋，可是幻覺很重，每一次都要從頭跟它解釋自己的狀況和這手牌，又花時間又費力，產出品質還不如預期。對他來說，AI 的幫助很有限，頂多偶爾討論幾個技巧而已。

我一邊聽一邊就清楚了：問題不在 AI，在用法。他卡在四件事上。免費版有額度限制；網頁版記憶不連貫、存不下來；不懂得管理 context，一個對話框聊到底，記憶就開始丟失跟混亂；最關鍵的是，網頁版的 AI 根本不認識他，沒有長期記憶，也讀不了檔案，沒辦法長期累積。
摸清楚他的真實狀況之後，我才動手，幫他客製一套「職業德撲玩家可以拿 AI 來成長」的教材。

設計思路

我不是撲克專家，所以我不幫他蓋房子。這點是整套教材的前提。卡牌和德州撲克他是行家，我是小白，頂多看過別人玩、聽過幾個名詞，但我擅長 AI。所以我先用 AI 快速把德撲的概念和流程摸過一輪，再透過訪談搞懂他賽前怎麼準備、賽中在想什麼、賽後怎麼復盤、平常怎麼學。
我給的是藍圖跟方向，不是一套幫他建好的系統。我建的系統很可能不符合他的真實場景，那不如把可能性畫出來，讓他自己去長。

核心對比：通用軟體 vs 客製化 agent。德撲玩家復盤，靠的是一套通用軟體去算每手牌的勝率，那是給所有人用的 general 答案。我告訴他，AI agent 能做到的是完全相反的事：依你個人的風格打造專屬的打法，再針對每一個對手的習慣設計進攻。一個是公版，一個是只屬於你的。這就是本地知識庫加上客製化的威力，也是兩者最大的分水嶺。

入門篇：先讓他換掉工具。第一份文件講最關鍵的一步：別再用網頁版的 chat，改用 AI agent。Agent 搭配本地資料庫（例如 Obsidian），才能操作檔案、長期記憶、做出各種 skill 陪練陪復盤，還能把網路上的資源、每一場賽局都存進個人資料庫，長時間累積。簡單說，入門篇就是告訴他 AI agent 怎麼當好他的撲克教練，給他幾條路徑：把撲克概念萃取成知識庫並內化、用 agent 做每一局的復盤和對手 profiling、一整套工具箱。

進階篇：給他想像力，不是給他系統。當他初步嚐到 agent 的甜頭、感受到它跟通用軟體的代差之後，我再給第二份。這次我教的不是「AI 怎麼幫你打牌」，而是架構、空間和想像力。
Claude Code 這類 agent 有哪些發展方向適合德撲玩家去長？我舉了對手情報庫、記憶庫、個人化教練、復盤指令、手牌分析這些例子，但全部的核心其實只圍繞一件事：context。我告訴他怎麼累積自己的 context，怎麼把它轉成飛輪，讓他越用越強。
我畫一張初步的藍圖，剩下的，他照著去跟他的 agent 討論需求、微調出屬於自己的內容。

教材架構

入門篇：用 AI agent 當你的撲克教練

  • 診斷：你不是不努力，是工具用錯了，網頁版 chat 給不了你要的東西
  • 關鍵一步：從網頁版 chat 換成 AI agent 加本地資料庫
  • 三條路徑：撲克概念萃取成知識庫、每一局用 agent 復盤、對手 profiling
  • 完整工具箱：對話式手牌復盤、Range 思考訓練、對手模擬對打、Tilt 管理
  • 核心對比：通用軟體算 general 勝率，agent 給你個人化打法
  • 限制說明：賽中使用是紅線

進階篇：Claude Code 的可能性藍圖

  • 給架構與想像力，不給做好的系統
  • 可能性地圖：對手情報庫、記憶庫、個人化教練、復盤指令、手牌分析
  • 所有方向的核心都是同一件事：怎麼累積你的 context
  • Context 飛輪：為什麼越用越強
  • 怎麼跟你的 agent 討論需求，自己長出個人化內容
教材節錄｜通用軟體 vs 客製化 agent

你現在用的軟體，算的是每手牌的勝率，那是一套給所有人的公版答案。AI agent 不一樣。它讀你過去的牌局、你的風格、你常犯的錯，給你的是只屬於你的打法調整。對手也是。把每個常遇到的對手建一份檔案，下次上桌前跟它說「看一下老王的檔案，幫我出今晚針對他的策略」，它就能依你累積的情報，給針對這個人的作戰計畫。公版軟體做不到這件事，這就是本地知識庫的威力。

教材節錄｜為什麼一定要換成 agent

網頁版的 AI 不認識你。每次開新對話，它都把你當陌生人，你得從頭解釋自己的狀況和這手牌，記憶存不下來，聊久了 context 還會亂掉。AI agent 把知識存在你電腦的檔案裡，不在它的記憶裡。你可以累積幾個月的對手 profile、概念筆記、漏洞追蹤，然後問它「看完我這三個月的記錄，我的核心問題是什麼」。它讀得到，因為那些都是你的檔案。

我畫的是一張藍圖，不是一棟蓋好的房子。路怎麼走、房子長成什麼樣，是他的事。

成果

分享完，他當場就訂閱了 Claude Code，開始用 AI Agent 建自己的本地知識庫和 AI 撲克教練。他跟我說，用起來比過去的網頁版好太多了，是跨世代的差距。過去那些幻覺、亂答、不記得他的問題，在 agent 上全部被解決了。

他也靠著這套不斷迭代、累積、復盤，在賽場上越打越順。
最具體的一次，是 agent 從他累積的牌局裡，抓出一個他反覆在犯、自己卻一直沒察覺的盲點。那是一個固定情境下的 leak，過去一個人對著數據看怎麼都看不出來，因為他不知道要往那裡找。
勝率短時間還看不到明確的成長，但他說最核心的從來不是勝率立刻漲幾趴，而是身邊終於有一個真正懂他的顧問。過去他只能一個人對著數據想，現在 agent 能給他全然不同的切角和思路，讓他用過去累積的 context 繼續往前走。

這堂課教會我的事

我不懂各行各業所有的知識。德州撲克我是小白，但這不妨礙我幫到一位世界賽亞軍。我擅長的，是用 AI 工具快速進入一個陌生領域，學會對方的用詞，然後用他的語言跟他溝通。

同樣是 AI agent 和 context 飛輪，我如果用通用、概念性的講法跟他說，他很難立刻連結回德州撲克。所以我全程用牌桌的語言、用卡牌對戰的邏輯講，所有案例、框架、累積，全部長在他的需求和場景上。
教學的核心是「對方需要什麼，我給什麼」，不是「我有什麼，我教什麼」。他需要的不是 AI 的理論和原理，是怎麼拿這個工具把牌打得更好。
我幫人的方式，是教他用好工具去提升自己的領域，而不是教他這個工具是怎麼造出來的。
